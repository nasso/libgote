gt_storage_t* gt_map_storage ( void(*)(void *item)  destroy_item)

Create a gt_storage_t that stores items in a map.

Items are stored in a hash map where the keys are the identifiers and the values are the items. Useful when there isn't many items with small identifiers or when they are very rare.

Create a gt_storage_t that stores items in a vector.

Items are indexed directly using their u64_t identifier. This is very efficient when most identifiers are close to zero and leave very few to no gaps between them.
